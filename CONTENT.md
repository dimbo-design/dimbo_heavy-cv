# Как менять контент

Весь контент живёт в одном файле — `js/content.js`. Ни один другой файл при
редактировании текстов трогать не нужно. Ниже — правила, при которых логика
не поедет.

## Устройство узла (NODES)

Каждый пункт навигации — объект в массиве `NODES`:

```js
{
  id: 'warehouse',                 // латиницей, уникальный, не менять после публикации
  pose: { x: 0.46, rotY: -0.52, scale: 0.9, dim: 0 },
  anchor: { x: 3.7, y: 1.4, z: 3.0 },
  label: { ru: 'склад ke × uzum', en: 'warehouse ke × uzum' },
  sub:   { ru: 'sla 72 → 100', en: 'sla 72 → 100' },
  panel: { ru: {...}, en: {...} },
}
```

### anchor — где узел висит вокруг силуэта

Сцена в «мировых» единицах: видимая плоскость зеркала ≈ 10.4 × 7.8,
центр — (0,0). **Жёсткие пределы досягаемости: |x| ≤ 3.9, |y| ≤ 3.0** —
дальше палец отражения физически не дотягивается (рука выходит из кадра
камеры и трекинг умирает). z — глубина 1.4–3.4 (больше z = ближе к зрителю,
подпись крупнее). Не ставь два якоря ближе ~1.5 друг к другу — фокус будет
дребезжать между ними.

### pose — куда отходит форма, когда глава открыта

`x` — доля полуширины экрана со знаком: **положительный x = форма уходит
вправо, контент открывается слева («кейсы»); отрицательный — зеркально
(«о человеке»)**. Сторона главы определяется только знаком `pose.x` —
ничего больше настраивать не нужно (скрим, колонка, шкала позиции и жест
закрытия вбок подстроятся сами). `rotY` — разворот (±0.3…0.8), `scale` —
размер (0.7–1.15), `dim` — притушить (0–0.5; ставь выше, когда глава
фотографийная и форма должна уступить сцену).

### label / sub

Всегда обе локали. label — до ~20 символов (это строка в mono, длинную
разорвёт визуально), lowercase — стилевая норма. sub появляется при фокусе.

## Блоки главы (panel.ru.blocks / panel.en.blocks)

`kicker` (строка-надзаголовок, lowercase) + `title` + массив `blocks`.
Типы блоков — рендерятся в порядке следования:

| блок | форма | зачем |
| --- | --- | --- |
| абзац | `{ p: 'текст' }` | основной текст |
| абзац с подзаголовком | `{ h: 'проблема', p: 'текст' }` | главы кейса |
| тихая ремарка | `{ aside: 'текст' }` | mono-приписка в конце |
| метрики | `{ metrics: [['72% → 100%', 'подпись'], …] }` | 2–4 строки, не больше |
| таймлайн | `{ timeline: [['даты', 'роль', 'место'], …] }` | опыт |
| ссылки | `{ links: [['https://…', 'подпись ↗'], …] }` | контакты, внешнее |
| фотополоса | `{ strip: [['assets/gallery/имя.jpg', 'подпись'], …] }` | галерея |

Правила:
- **Каждая правка — в обеих локалях.** Структура блоков ru и en должна
  совпадать по порядку и типам (переводи текст, не перестраивай).
- **Ссылка на pdf** (`… .pdf`) автоматически становится «жестовой»
  (скачивается удержанием руки). Внешние ссылки автоматически получают
  `target=_blank` и работают только мышкой — это осознанное правило.
- **strip**: все фото одной главы попадают в один полноэкранный просмотр
  (лайтбокс) в порядке следования. Кулак/тап по фото открывает именно его.
- Не ставь два strip подряд без текста между ними — жест «какую полосу
  тяну» станет неоднозначным.

## Фотографии

- Кладём в `assets/gallery/`, **jpeg**, длинная сторона ≤ 1600–1800 px,
  вес ≤ ~450 КБ. Конверсия на маке:
  `sips -Z 1600 -s format jpeg -s formatOptions 84 вход.png --out assets/gallery/имя.jpg`
- Имена — смысловые латиницей (`release-boxes.jpg`), не `IMG_1234`.
- Подпись обязательна: она и figcaption в полосе, и подпись в лайтбоксе.

## Добавить / убрать узел

1. Скопируй существующий объект узла, поменяй `id`, тексты, фото.
2. Подбери свободный `anchor` в пределах досягаемости (см. выше) — глянь
   на занятые места, узлы не должны толпиться.
3. Реши сторону: кейс → `pose.x` положительный, «о человеке» → отрицательный.
4. Всё. Регистрация нигде больше не нужна — сцена, фокус, шкала и жесты
   подхватывают массив целиком.

Разумный диапазон — **5–8 узлов**: меньше — пусто, больше — фокус от руки
начинает промахиваться (узлы толпятся).

Убрать узел — просто удалить объект из массива.

## Строки интерфейса

Объект `UI` там же: приглашение (`invite`, вариант `hReturn` для
вернувшихся), отказ от камеры, мобильный тупик, телеметрия, мини-контакты
(`contactsMini` — используются в denied/failed/mobile). Правило то же:
обе локали сразу.

## Проверка после правок

```bash
python3 -m http.server 8000   # модули не работают с file://
```

- `http://localhost:8000/?fresh` — сайт встречает как незнакомца
  (сбрасывает память возвращения и язык).
- `?debug` (или ⌥D) — телеметрия и журнал жестов; ⌥C / кнопка — скопировать.
- Пройди: узел открылся → скролл → фото → листание → закрытие; переключи
  язык — глава перерисуется на месте.

Чего не делать: не менять `id` опубликованных узлов (ломает привычку и
якоря), не класть видео в полосы (не поддержано), не убирать `download`
у pdf-ссылки, не писать заглавными label'ы, не ставить якоря за пределами
досягаемости.
