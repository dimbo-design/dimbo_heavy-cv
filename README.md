# Дмитрий Тогулев — живой интерфейс

Портфолио, в котором камера — основной способ ввода. Страница существует как
спящее поле частиц; когда камера замечает человека, поле собирается в его
объёмное отражение, и вокруг силуэта проявляется контент. Руки видны
интерфейсу так же, как и лицо: навигация построена на жестах.

## Как это устроено

- [Depth Anything V2 Small](https://huggingface.co/onnx-community/depth-anything-v2-small)
  (ONNX) считает карту глубины по кадрам веб-камеры — в браузере, через
  **Transformers.js** (WebGPU, фолбэк на WASM). Ничего не уходит с устройства.
- **MediaPipe Hand Landmarker** (основной поток, GPU) ведёт кисти рук — из
  него собран жестовый слой: кольцо-курсор, захват щипком или кулаком, тап.
- **Three.js**: ~23 000 частиц, один шейдер покрывает все состояния — загрузка
  модели материализует поле (аккреция), присутствие собирает его в форму,
  открытая глава отводит форму в сторону.
- Глубина живёт в **Web Worker**-е; руки — на основном потоке (канонический
  путь MediaPipe, ~5 мс/кадр). Телеметрия честно показывает состояние рук.
- Присутствие, близость и положение головы — эвристики прямо по карте глубины
  (доля переднего плана, компактность пятна, центроид, движение), без третьей модели.

## Словарь жестов

Минимальный и бесконфликтный: одно движение — одно значение, каждое действие
показывает себя кольцом до срабатывания и обратимо. Сцена стабильна — рука
двигает только собственное кольцо, объёмная форма следует лишь за головой.

| жест | действие |
| --- | --- |
| появиться в кадре | интерфейс просыпается, узлы проявляются |
| движение руки / пальца | кольцо-курсор, фокус на ближайшем узле |
| задержать спокойную руку на узле | кольцо заряжается → глава открывается |
| захват (щипок или кулак) + движение | утащить — ось запирается: горизонталь над полосой листает её, вертикаль скроллит главу |
| сжать и держать неподвижно | кольцо заряжается → закрыть главу / лайтбокс |
| короткий щипок | тихий «клик»: фото → лайтбокс, узел → глава |
| выйти из кадра | всё закрывается, отпечаток медленно тает |

Фото раскрываются в лайтбокс FLIP-анимацией — снимок вырастает из своего
места в полосе и возвращается туда же; листание — направленный перенос
(тянуть захватом, тап по половине кадра, колесо, стрелки); за последним
кадром — возврат в главу. Полоса наклоняется в сторону движения.

Внешние ссылки — намеренно только мышкой (взял мышь — вернулся в привычный
веб). Единственное исключение — pdf-резюме: задержи на нём руку, кольцо
зарядится, и файл скачается сам.

Сайт помнит возвращение: во второй визит приглашение звучит иначе, интерфейс
раскрывается быстрее. Когда человек уходит из кадра, отпечаток его формы
медленно расплывается обратно в дрейф — и быстро собирается при возвращении.

Тихие дублёры (нигде не анонсируются): клик по узлу, колесо, drag мышью,
`Esc`. `Alt+D` — телеметрия сигналов и лог распознанных жестов; пороги — в
`js/config.js`.

## Запуск локально

Сборки нет. Нужен любой статический сервер (модули и воркеры не работают с `file://`):

```bash
python3 -m http.server 8000
# → http://localhost:8000
```

Первый запуск скачивает модели (~45 МБ суммарно), дальше — кэш браузера.

## Деплой на GitHub Pages

Репозиторий: `dimbo-design/dimbo_heavy-cv`. После пуша:
Settings → Pages → Source: `main` / root. Файл `.nojekyll` на месте,
камера требует HTTPS — GitHub Pages подходит.

## Производительность

Каденс инференса адаптивный (`js/config.js → cadence`): глубина 20 к/с при
присутствии (вход модели уменьшен до 280×210 — на треть быстрее), 11 к/с при чтении главы, 6 к/с когда в кадре никого; руки — 30 к/с только при присутствии. Пиксель-рейшо ограничен 1.5, в пустых состояниях рендер идёт
на половинной частоте. Кроссфейд карт глубины в шейдере делает низкий каденс
незаметным.

## Поддержка

Десктопные Chrome / Edge / Arc (WebGPU) — основной путь; Safari 18+ и Firefox —
через WASM (медленнее «мысль», интерфейс жив). Мобильные устройства получают
спроектированный тупик с контактами.

## Структура

```
index.html          разметка всех состояний
css/main.css        весь стиль (Fixel Display + IBM Plex Mono)
js/config.js        все настройки и пороги ощущений
js/content.js       контент RU/EN, якоря узлов, рендер глав
js/main.js          стейт-машина, сигналы, жесты → смысл, DOM-хореография
js/scene.js         поле частиц (Three.js, один шейдер, позы формы)
js/depth.js         камера + пампинг кадров в depth-воркер
js/depth-worker.js  Depth Anything V2 + статистика присутствия
js/hands.js         MediaPipe Hand Landmarker → компактная геометрия кисти
js/gestures.js      курсор, захват/тап/тычок/бросок из геометрии
assets/fonts/       Fixel Display (otf)
assets/gallery/     фотографии кейсов и настолок
assets/             фавиконка, og-картинка, резюме (pdf)
```
