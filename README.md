# Дмитрий Тогулев — живой интерфейс

Портфолио, в котором камера — основной способ ввода. Страница существует как
спящее поле частиц; когда камера замечает человека, поле собирается в его
объёмное отражение, и вокруг силуэта проявляется контент.

**Как это устроено**

- [Depth Anything V2 Small](https://huggingface.co/onnx-community/depth-anything-v2-small)
  (ONNX) считает карту глубины по каждому кадру веб-камеры — прямо в браузере,
  через **Transformers.js** (WebGPU, фолбэк на WASM). Ничего не уходит с устройства.
- **Three.js**: 30 000 частиц, один шейдер покрывает все состояния —
  загрузка модели материализует поле (аккреция), присутствие собирает его в форму.
- Инференс живёт в **Web Worker** — рендер держит 60 fps независимо от модели.
- Присутствие, близость и положение головы вычисляются эвристиками прямо по
  карте глубины (доля переднего плана, компактность пятна, центроид, движение) —
  без второй ML-модели.

**Управление присутствием**

- появление в кадре → интерфейс просыпается, имя и узлы контента проявляются сами;
- горизонтальное движение головы → фокус скользит по узлам;
- наклониться к экрану → сфокусированный узел раскрывается в панель;
- откинуться назад / уйти из кадра → панель закрывается, интерфейс засыпает.
- тихие дублёры (нигде не анонсируются): клик по узлу, `Esc`, клик мимо панели.
- `Alt+D` — отладочная телеметрия сигналов (пороги — в `js/config.js`).

## Запуск локально

Сборки нет. Нужен любой статический сервер (модули + воркер не работают с `file://`):

```bash
cd interation_web
python3 -m http.server 8000
# → http://localhost:8000
```

Первый запуск скачивает модель (~40 МБ), дальше она живёт в кэше браузера.

## Деплой на GitHub Pages

```bash
git remote add origin git@github.com:<user>/<repo>.git
git push -u origin main
```

Затем: Settings → Pages → Source: `main` / root. Файл `.nojekyll` уже на месте.
Камера требует HTTPS — GitHub Pages подходит.

## Поддержка

Десктопные Chrome / Edge / Arc (WebGPU) — основной путь; Safari 18+ и Firefox
работают через WASM-фолбэк (медленнее «мысль», но интерфейс жив). Мобильные
устройства получают спроектированный тупик с контактами.

## Структура

```
index.html          разметка всех состояний
css/main.css        весь стиль
js/config.js        все настройки и пороги ощущений
js/content.js       контент RU/EN + якоря узлов в 3D
js/main.js          стейт-машина, сигналы, DOM-хореография
js/scene.js         поле частиц (Three.js, один шейдер)
js/depth.js         камера + пампинг кадров в воркер
js/depth-worker.js  инференс Depth Anything V2 + статистика присутствия
assets/             фавиконка, og-картинка, резюме (pdf)
```
